# ADR 0004 — RBAC

* Status: accepted
* Date: 2026-04-12

## Context

We need to differentiate humans from machines, and within humans
distinguish admins from researchers from read-only viewers.

## Decision

A simple Postgres `role` enum (`admin`, `researcher`, `viewer`,
`service`) with route-level dependency injection (`require_role(...)`)
and per-API-key scope tags (`read`, `write`, `ingest`, `notify`).

* Refresh tokens persist for 14 days; access tokens for 1 hour.
* API keys are stored as SHA-256 digests; the prefix `afk_` is shown
  to the user only at creation time.

## Consequences

* No third-party identity provider needed for the MVP.
* Adding SAML / OIDC in the future means swapping the
  `auth/token` endpoint for an OAuth2 callback flow without touching
  any other service.
